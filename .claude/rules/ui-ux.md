# UI/UX Rules

## Screen Structure

### ボタンビュー（デフォルト画面）
- 習慣名を上部に常時表示
- 達成ボタンを大きく中央に配置（片手で押しやすいサイズ）
- 連続日数を小さく表示
- **横スワイプ** → 他の習慣のボタンビューに切替
- **下スワイプ** → カレンダービューに遷移

### カレンダービュー
- 習慣名を上部に常時表示
- 草グラフ（SVG）+ 表示期間切替タブ
- 過去日付へのさかのぼり記録もここから
- **上スワイプ** → ボタンビューに戻る

### アクティビティ一覧
- 全カレンダーをリスト形式で表示
- ミニグラフ + 連続数

### 習慣管理
- 作成・編集・アーカイブ・完全削除
- 色選択、アニメーションパターン選択・プレビュー

## Swipe Implementation

- 横スワイプ（習慣切替）と縦スワイプ（ボタン↔カレンダー）が共存する
- ジェスチャー判定のしきい値を慎重に設定すること（斜めスワイプの誤認識に注意）
- スクロールとスワイプの競合を防ぐこと

## Theme

- カレンダーごとの色をCSS変数で動的切替
- ダークモード対応（next-themes）
- スマホファースト（モバイル最優先のレスポンシブ）
